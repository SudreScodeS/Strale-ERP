# Safe Project Edit

Esta skill orienta alterações seguras no projeto Elitium ERP.

## Antes de alterar

Sempre faça:

1. Entender a estrutura do projeto.
2. Identificar stack e dependências.
3. Localizar arquivos relevantes.
4. Explicar o plano de alteração.
5. Listar riscos.
6. Evitar mexer em muitos arquivos ao mesmo tempo.

## Durante a alteração

- Alterar o menor número possível de arquivos.
- Preservar componentes existentes.
- Não remover código sem justificar.
- Evitar grandes refatorações.
- Não adicionar dependências sem necessidade.
- Não mudar identidade visual sem motivo.
- Não criar páginas novas sem necessidade.
- Não quebrar componentes existentes.
- Não alterar arquitetura sem explicar impacto.
- Não reescrever tudo só para "melhorar".

## Depois da alteração

Sempre entregar:

- arquivos alterados;
- resumo do que mudou;
- como testar;
- possíveis pontos de atenção;
- próximos passos recomendados.

## Regras para o Elitium ERP

O projeto deve continuar parecendo um ERP/SaaS profissional.

Priorize:

- estabilidade;
- clareza;
- organização;
- visual premium;
- componentes reutilizáveis;
- responsividade em 1920x1080;
- viabilidade para TCC.

Evite:

- mudanças gigantes sem necessidade;
- refatorações arriscadas;
- dependências novas sem justificativa;
- design genérico de IA;
- alterações que quebrem o projeto.
