# ERP Product AI

Você deve ajudar a criar ou melhorar uma IA de Produtos integrada ao Elitium ERP.

## Objetivo

A IA de Produtos deve parecer uma funcionalidade real, útil e viável dentro de um ERP/SaaS.

Ela não deve parecer apenas um texto bonito na landing page.

## Funcionalidades recomendadas

- Analisar produtos com baixa saída.
- Alertar estoque crítico.
- Sugerir reposição.
- Identificar produtos parados.
- Sugerir ajuste de preço.
- Sugerir margem ideal.
- Destacar produtos mais vendidos.
- Gerar insights comerciais.
- Simular respostas de assistente com base em dados mockados.
- Criar cards de recomendação inteligente.
- Conectar visualmente Produtos, Estoque, Pedidos e Financeiro.

## Regras

- Começar com dados mockados se não houver backend pronto.
- Não integrar API externa sem autorização.
- Não prometer machine learning real se for apenas simulação.
- Deixar claro no código quando os dados forem mockados ou demonstrativos.
- Priorizar uma UX convincente para apresentação do TCC.
- A IA deve dar recomendações práticas, não respostas genéricas.
- A IA deve parecer integrada ao ERP, não um chatbot solto.

## Tom da IA

A IA deve responder como uma assistente operacional:

- direta;
- útil;
- baseada em dados;
- com recomendações práticas;
- sem parecer chatbot genérico.

## Exemplos de insights

- "Produto com estoque alto e baixa saída nos últimos 30 dias."
- "Produto próximo do estoque mínimo."
- "Produto com boa margem, mas baixa exposição nos pedidos."
- "Produto com alta saída e necessidade de reposição."
- "Produto com margem abaixo da média da categoria."

## Critério de qualidade

Antes de finalizar qualquer melhoria na IA de Produtos, verifique:

- A funcionalidade parece real?
- Os dados fazem sentido?
- A IA ajuda na tomada de decisão?
- A interface parece integrada ao ERP?
- O resultado é viável para TCC?
- O usuário consegue entender rapidamente o valor da IA?
